CouchPilot 会保存本地聚合事实，让键位决策基于真实使用，而不是凭感觉。报告要回答的是：每个 App 里哪些按键真正有用，某个组合键是否可能难按，哪些键位没有获得使用机会，以及哪些映射值得做改前/改后实验。它会记录证据，但不会自动改键位。

## 会记录什么

每次有意义的手柄交互会累计当天日期、前台程序的可执行文件名（例如 `ChatGPT.exe` 或 `chrome.exe`）、当前方案、物理按键、规范化手势、真正命中的绑定来源、逻辑动作、派发结果，以及当前有效键位策略的稳定版本号。App 方案回退到默认方案时也会保留真实来源，这样既不会把次数算到错误的映射上，也不会把改键前后的数据混在一起。

程序还会提取一些保护隐私的粗粒度 tracing 事实：

- 物理按键是否发生重叠，以及最终命中了已绑定、已禁用、未绑定还是回退手势；
- 按住时长和相邻动作间隔的大致区间，而不是精确时间；
- 交互、语音编辑、窗口切换和连续重复等过程以及它们的结束方式；
- 看起来可能是“撤回或纠正”的短间隔后续模式。

快照还会保存当前可用控制和绑定清单。配合各 App 的交互机会，报告可以区分“已经禁用”、“有机会但尚未使用”和“根本没有获得使用机会”。

“派发成功/派发失败”只表示 CouchPilot 是否成功向系统发出动作，不能证明用户的真实任务或意图是否成功。

## 怎样理解这些线索

**组合键近失误**指的是：物理按键出现了像组合键的重叠，但最终没有按该组合键解析。这可能表示触发时机或手指距离不舒服，也可能只是故意的单键动作或偶然重叠；它不能证明用户当时就想按那个组合键。由于 LT/RT 同时承担鼠标精度或加速功能，发生在指针或摇杆活动上下文中的探测会单独标为“歧义”，只展示原始数据，不进入改键建议的有效分母。

**疑似纠正**指的是：后续的粗粒度模式（例如很快向反方向操作）看起来像是在纠正。它适合用来找待调查的候选键位，但不能确认前一次操作就是错误。改键前应结合 App、使用天数、交互机会和策略版本一起看。

报告不会根据孤立事件给建议。组合候选必须在同一策略、App 和场景下拥有足够的有效分母、达到最低占比，并分布在至少两个活跃日；疑似纠正和快速重复也必须同时满足同源转移分母、出现次数、占比和跨日要求。具体门槛会直接显示在报告中。

## 不会记录什么

本地按键统计文件不会记录你输入的文字，不会安装全局键盘监听，也不会保存窗口标题、完整进程路径、手柄编号、鼠标坐标或精确的按键时间线；它只保留可执行文件名，不保留程序所在位置，并把粗粒度时间区间等事实按天聚合。记录开启时，程序压缩数据并只保留 90 天的滚动窗口；关闭记录后，已有本地文件会冻结保留，直到你手动删除。数据绝不会上传或通过网络发送。普通诊断日志与统计文件相互独立，开启 `--verbose` 时日志仍可能包含用于排障的动作级信息。

## 在哪里查看

右键点击 CouchPilot 托盘图标，选择 **查看按键报告**，即可打开容易阅读的本地报告；也可以运行下面的命令，在终端查看同一份实时汇总：

```powershell
.\bin\couchpilot.exe usage
```

文件位于 `config.json` 旁边：

```text
usage/
  usage-v1-report.html
  usage-v1.snapshot.json
  usage-v1.wal.jsonl
```

快照文件保存稳定的聚合视图，有新数据时大约每分钟自动追平一次；JSONL 文件保存最近一小段、有上限的崩溃恢复日志。两者合在一起才是当前完整记录，CouchPilot 会自动压缩日志。目录里还可能出现内部恢复副本 `usage-v1.snapshot.json.bak`。程序运行时不要手动编辑这些文件。

## 一分钟验收

1. 打开报告，先记住任意一个按键当前的次数。
2. 把这个键分别按下三次，不要一直按住。
3. 等待约 10 秒（最多 15 秒），然后刷新或重新打开报告。次数应正好增加 3，并且该键会从“当前键位策略尚未观察到”中消失。
4. 再用已定义组合键（例如 `LT+RB`）重复一次。组合键次数应增加，持续按住和松开都不应额外计数。
5. 正常操作时，“派发失败”和“丢弃”应保持为 0。

这只验收记录是否准确。组合键近失误和疑似纠正需要一段正常使用才有参考价值，而且永远只是线索，不是已确认的意图或错误。报告不会自动修改键位。

## 关闭记录

本地聚合记录默认开启。在 `config.json` 中加入下面的设置并重启 CouchPilot：

```json
{
  "local_usage_stats_enabled": false
}
```

关闭后不会继续增加计数，也不会自动删除已有数据；已有报告仍可从托盘打开。如需清除，请先停止 CouchPilot，再删除对应的 `usage` 文件夹。这个开关只控制本地聚合记录，绝不会开启网络上传。
